AI-Powering Dreams: Part 2

AI empowers me to dream big and act bold. In just a short time, AI helped me migrate my blog theme, optimize SEO, rewrite GitHub Actions, and integrate social media links. Complex tasks I once avoided are now effortless with AI collaboration!
March 26, 2026  · eatmoreduck

Introduction#

In my previous post AI-Powering Dreams, I documented my experience using Claude Code with glm5 to build a cross-platform skill management tool.

This time, I set my sights on my own blog. With AI assistance, I completed a series of improvements in just 12 hours that I previously found too complex or troublesome to tackle.

Transformation Checklist#

Here’s what was accomplished:

Theme Migration#

Switched from my old theme to PaperMod, which offers:

  • Clean and elegant design that matches my aesthetic
  • Responsive design, mobile-friendly
  • Built-in dark mode
  • Excellent SEO support

The migration involved rewriting numerous configuration files:

  • All files under config/_default/
  • Homepage, archives, links pages
  • Custom style files

Honestly, if I had configured this manually, just researching each configuration option would have taken considerable time. But with the assistance of Claude Code + glm4.7, the entire process was much smoother.

Social Media Integration#

Added multiple social media icons and links:

  • WeChat (sidebar QR code)
  • GitHub
  • Bilibili
  • YouTube

These icons weren’t taken directly from the theme. I found official SVGs from Tabler Icons myself or had AI help find them, then manually added them to the project. There were some back-and-forth moments—for instance, the WeChat SVG would get corrected and then changed back incorrectly. The inconsistency was frustrating until I finally added this rule directly to CLAUDE.md:

## The WeChat social media SVG is fixed. Do not modify it!

Share Button Enhancement#

Enhanced article sharing functionality to support:

  • WeChat
  • Moments
  • Zhihu
  • Twitter
  • Facebook
  • LinkedIn

Although the Zhihu share button still has a bug, I’m quite satisfied with the current result—I’d give it a 90 out of 100.

Comprehensive SEO Optimization#

This was something I’d wanted to do for ages but found too complex:

Structured Data:

  • BlogPosting
  • FAQPage
  • HowTo
  • VideoObject
  • Review
  • SoftwareApplication

Verification Configuration:

  • Completed Google Search Console verification code

AI Discoverability:

  • Added llms.txt file to help AI tools better understand blog content

GitHub Actions Performance Optimization#

My blog deployment uses GitHub Actions, but previous build efficiency was somewhat low.

Optimizations:

  • Upgraded to Node.js 24
  • Added Hugo CLI caching
  • Added Git Submodule caching
  • Optimized workflow step structure

Build time significantly decreased.

Tag System Refactoring#

Previously, my blog used both categories and tags, but categories were barely used and their functionality overlapped.

Refactoring:

  • Removed categories field from all articles
  • Added tags to articles with empty tags
  • Fixed typos (ags → tags)
  • Simplified article creation workflow (Obsidian QuickAdd script optimization)

Analytics Integration#

In addition to existing Google Analytics, added Cloudflare Web Analytics:

Advantages:

  • Privacy-friendly, no cookies
  • Lightweight, doesn’t affect page load speed
  • Clean, intuitive interface

Links Page Redesign#

Redesigned the friendship links page:

  • Added social media links (Zhihu, Bilibili, YouTube, GitHub)
  • Added RSS subscription link
  • Optimized link exchange copy
  • Enabled comments (powered by Giscus)

Although there are no friendship links yet, I really hope to make more friends and add more links~

How I Worked#

This transformation was done using Claude Code. The workflow was roughly:

  1. State Requirements: Tell Claude what I want to do
  2. Analyze Options: Claude analyzes the codebase and provides suggestions
  3. Confirm Execution: After I confirm, Claude directly modifies the code
  4. Verify Results: Push code and check the results

The entire process didn’t require me to write any code manually. I only needed to:

  • Be clear about what I want
  • Judge and confirm Claude’s suggestions
  • Verify if results meet expectations

Thoughts After Intensive AI Use#

Dare to Dream, Dare to Act#

Previously, when I saw cool features on other blogs, I wanted to add them but felt they were too complex:

  • Theme configuration with tons of parameters I didn’t understand (felt like spending time on decoration when I should focus on content production)
  • SEO structured data that I knew nothing about (after this experience, I at least know what points need to be considered)
  • GitHub Actions optimization, afraid of breaking things (this is crucial—I remember how hard it was to fix broken deployment actions in the past)

Now with AI collaboration, these are no longer barriers. Just state your requirements, and AI will help implement them.

Dramatically Reduced Time Costs#

This transformation took about 12 hours, but actual operation time was only a few hours:

  • Most time was spent waiting for GitHub Actions deployment
  • Actual thinking and decision-making time was limited

If I had researched from scratch:

  • Researching theme configuration: could take days
  • Learning SEO best practices: could take a week
  • Optimizing GitHub Actions: might require repeated debugging

AI Is the Co-Pilot#

AI won’t replace you, but will make you stronger:

  • You still need to know what you want
  • You need to judge whether AI’s suggestions are reasonable
  • You need to verify if results meet expectations

AI is the co-pilot. You are the captain.

Next Steps#

I’ve set a flag for myself: publish at least one blog post per week.

With AI’s assistance, I feel confident this flag is sustainable.

Conclusion#

AI empowers me to dream big and act bold. Tasks that once seemed complex, troublesome, or undesirable can now be completed with minimal effort, thanks to AI collaboration.

This is not just an efficiency improvement—it’s an expansion of possibilities.

AI powers every dream. This is not empty talk.
