AI-助力每一个梦想之二

AI让我敢想敢做，花了一点点时间，AI帮我换掉了博客主题，优化了SEO，重写了GitHub Actions，加入了各种社交媒体链接。以前不敢想的复杂操作，现在有了AI协作，轻松搞定！
March 26, 2026  · eatmoreduck

前言#

在上一篇 AI-助力每一个梦想 中，我记录了用 Claude Code 配合 glm5 开发跨平台技能管理工具的经历。

这一次，我把目标对准了自己的博客。在12小时内，在 AI 的协助下，我完成了一系列以前不敢想或者觉得太麻烦的改造工作。

改造清单#

先看一下这次完成的改造内容：

主题更换#

从旧主题更换到 PaperMod，这个主题：

  • 简洁优雅，符合我的审美
  • 响应式设计，移动端友好
  • 内置暗色模式
  • 良好的 SEO 支持

更换过程涉及大量配置文件的重写：

  • config/_default/ 下的所有配置文件
  • 首页、归档、友情链接等页面内容
  • 自定义样式文件

说实话，如果是自己手动配置，光是研究各个配置项的作用就要花不少时间。但在 cc+glm4.7 的协助下，整个过程顺畅了很多。

社交媒体集成#

新增了多个社交媒体图标和链接：

  • 微信（侧边栏二维码）
  • GitHub
  • 哔哩哔哩
  • YouTube

这些图标不是直接用主题自带的，而是我自己从 Tabler Icons 找的官方 SVG，或者是让 AI 给我找，然后手动添加到项目中，虽然经历过多次的反复，比如微信的 SVG 改对了又给我换错，存在不稳定的情况，但最后我把这条规则直接写入 CLAUDE.md 得以解决

## 微信social media的svg已经固定，不要做改动!

分享按钮优化#

增强了文章分享功能，支持：

  • 微信
  • 朋友圈
  • 知乎
  • Twitter
  • Facebook
  • LinkedIn 虽然 知乎的分享还有 bug，但目前这个结果我相当满意了，能给到 90 分

SEO 全面优化#

这部分是我之前一直想做但觉得太复杂的：

结构化数据：

  • BlogPosting
  • FAQPage
  • HowTo
  • VideoObject
  • Review
  • SoftwareApplication

验证配置：

  • Google Search Console 验证码补全

AI 可发现性：

  • 添加 llms.txt 文件，让 AI 工具更容易理解博客内容

GitHub Actions 性能优化#

博客部署使用 GitHub Actions，之前每次构建效率有点低，。

优化内容：

  • 升级到 Node.js 24
  • 添加 Hugo CLI 缓存
  • 添加 Git Submodule 缓存
  • 优化 workflow 步骤结构

构建时间明显缩短了。

标签系统重构#

之前的博客同时使用 categories 和 tags，但实际上 categories 几乎没用，而且两者功能重复。

重构内容：

  • 删除所有文章的 categories 字段
  • 为空标签文章补充标签
  • 修复拼写错误（ags → tags）
  • 简化文章创建流程（Obsidian QuickAdd 脚本优化）

数据分析集成#

除了已有的 Google Analytics，新增了 Cloudflare Web Analytics：

优势：

  • 隐私友好，不使用 Cookie
  • 轻量级，不影响页面加载速度
  • 界面简洁直观

友情链接页面#

重新设计了友情链接页面：

  • 添加社交媒体链接（知乎、B站、YouTube、GitHub）
  • 添加 RSS 订阅链接
  • 优化友链交换文案
  • 启用评论功能（基于 Giscus）

虽然目前没有一个友链，但还是很希望能多交朋友，多多添加友链～

工作方式#

这次改造使用的是 Claude Code，工作流程大概是：

  1. 提出需求：告诉 Claude 我想要做什么
  2. 分析方案：Claude 分析代码库，给出建议
  3. 确认执行：我确认后，Claude 直接修改代码
  4. 验证结果：推送代码，查看效果

整个过程不需要我手动写任何代码，只需要：

  • 明确自己想要什么
  • 对 Claude 的建议进行判断和确认
  • 验证结果是否符合预期

高强度使用 AI 之后，我有很多想法#

敢想敢做#

以前看到别人博客有各种酷炫功能，自己想加但总觉得太复杂：

  • 主题配置一大堆参数，看不懂（感觉像是花时间装修，但是实际应该把时间花在内容产出上）
  • SEO 结构化数据，完全不会（经过这次了之后，起码知道哪些点需要考虑到）
  • GitHub Actions 优化，怕改坏了（这个是重中之重，想当初废了很大劲该坏了很多次部署的 action）

现在有了 AI 协作，这些都不是障碍。只要说出需求，AI 就能帮你实现。

时间成本大幅降低#

这次改造大概花了 12 小时，但实际操作时间可能只有几个小时：

  • 大部分时间在等待 GitHub Actions 部署
  • 真正的思考和决策时间很有限

如果是自己从零开始研究：

  • 研究主题配置：可能要几天
  • 学习 SEO 最佳实践：可能要一周
  • 优化 GitHub Actions：可能要反复调试

AI 是副驾驶#

AI 不会替代你，但会让你变得更强：

  • 你仍然需要知道想要什么
  • 你需要判断 AI 的建议是否合理
  • 你需要验证结果是否符合预期

AI 是副驾驶，你是机长。

下一步#

我也给自己立了一个 flag：每周至少产出一篇博文。

借助 AI 的力量，我觉得这个 flag 是可以坚持的。

AI 让我敢想敢做。以前觉得复杂、麻烦、不想做的事情，现在有了 AI 协作，花一点点时间就能完成。

这不仅是效率的提升，更是可能性的拓展。

AI 助力每一个梦想，这不是空话。
