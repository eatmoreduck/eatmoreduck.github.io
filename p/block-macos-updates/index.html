屏蔽 macOS 26 更新（2026 亲测有效，消除小红点）

2026 年亲测有效的 macOS 26（Tahoe）屏蔽更新方法：先关闭自动更新检测，再通过 hosts 拦截更新服务器，最后用配置描述文件（mobileconfig）彻底隐藏 macOS 升级提示并消除烦人的小红点。适用于不想升级、老机器怕变卡、或需要稳定开发环境的用户，附恢复更新的方法。

为什么要屏蔽更新#

macOS 26 出新之后，隔三岔五的弹窗更新，总让我担心手滑点到「立即更新」，虽然有二次确认。但最近更新频繁，我想找个一劳永逸的办法。

我想要屏蔽更新的几个原因：

  • 强迫症看着那个红点难受
  • 新系统老机器带不动，怕更新了变卡
  • 工作环境需要稳定，不想瞎折腾，更新后担心开发环境炸掉
  • macOS 26 的液态玻璃风格我很讨厌，同时也担心更新后部分 app 不能匹配

遂决定升级到 macOS 15.7.3 之后，彻底屏蔽 macOS 更新，同时消除小红点

下面的方法在 2026 年的 macOS 26（Tahoe）上仍然可用，但说实话，越新的系统对更新提示的「执念」越深。hosts 拦截能让更新下载失败，却不一定能干净地让小红点消失。所以我把方案分成两个层次：先上 hosts 兜底，再用配置描述文件做彻底屏蔽（这是 2026 年最推荐、最省心的做法）。

第一步：UI屏蔽软件更新及自动下载#

  1. 打开「设置」-「通用」-「软件更新」
  2. 点击右边的 ⓘ 图标
  3. 把「自动保持我的 Mac 最新」关掉

第二步：修改 hosts 屏蔽更新#

这一步的操作是为了禁止连接到 macOS 更新服务器，编辑 hosts 文件：

sudo vim /etc/hosts

在文件末尾加入以下几行：

127.0.0.1 swscan.apple.com
127.0.0.1 swdist.apple.com
127.0.0.1 swdownload.apple.com
127.0.0.1 swcdn.apple.com
127.0.0.1 updates-http.cdn-apple.com
127.0.0.1 updates.cdn-apple.com
127.0.0.1 gdmf.apple.com
127.0.0.1 xp.apple.com
127.0.0.1 mesu.apple.com

然后刷新 DNS 缓存：

sudo dscacheutil -flushcache
sudo killall -HUP mDNSResponder

第三步：消除小红点#

完成上一步之后，依然能看到 macOS 26 的更新提示，但点击更新会失败。此时先点击更新，在弹出的界面不要点击「检测更新」，然后按 command+option+esc 打开「强制退出」窗口，将「系统设置」强制退出，小红点就会消失

关于 hosts 方法的局限#

需要说明的是，从 macOS 15（Sequoia）到 26（Tahoe），Apple 把更新检测和通知做得越来越「顽固」。上面的 hosts 屏蔽可以让升级包下载不下来，但小红点和「升级到 macOS Tahoe」的提示有时还是会冒出来，尤其是系统后台守护进程定时检查之后。如果你发现红点三天两头又回来了，那就直接用下面第四步的配置描述文件，一劳永逸。

第四步（2026 推荐）：用配置描述文件彻底屏蔽升级#

这是 2026 年社区（macsysadmin、Jamf、Daring Fireball 等）一致推荐的做法，也是我个人认为最省心的方案。原理是给系统装一个「延迟软件更新」的描述文件，让 macOS 26 的升级入口在「软件更新」界面里直接消失，连红点都不给你留。个人 Mac 不需要企业 MDM，自己做一个 .mobileconfig 装上就行。

核心是 Restrictions（com.apple.applicationaccess）这个 payload 里的两个键：

  • forceDelayedMajorSoftwareUpdates = true：隐藏大版本升级（也就是 macOS 26）
  • managedInstallDelay = 90：延迟天数，最大 90 天

做一个描述文件#

不需要从零手写 XML，用免费的 iMazing Profile Editor 或 Apple Configurator（App Store 可下）新建一个 profile，添加 General → Restrictions，勾上强制延迟主版本更新，天数填 90，导出成 .mobileconfig 文件即可。

安装描述文件#

  1. 双击导出的 .mobileconfig 文件，系统会提示「已下载描述文件」
  2. 打开「设置」-「隐私与安全性」，下拉到底部找到「描述文件」，点击进入
  3. 选中刚才的描述文件，点击「安装」，按提示输入密码确认

装完回到「软件更新」看一眼，原本的「升级到 macOS Tahoe」入口和红点就都不见了。这比 hosts 折腾舒服多了。

注意事项#

  • 90 天上限：延迟最多只能设 90 天，到期前重新安装一次新 profile 就能续上，长期不升级的话设个日历提醒
  • 手动安装的限制：macOS 26 上手动安装的描述文件，部分 Restrictions 键的覆盖力不如 MDM 受管设备，但对隐藏 macOS 大版本升级这一项是够用的
  • 想升级时：把描述文件删掉（描述文件界面里点「移除」），稍等片刻就能重新看到更新入口

怎么恢复#

哪天想更新了，按你当初用的是哪种屏蔽方式来恢复：

  • hosts 方式：把 /etc/hosts 里相关行注释掉或删掉，刷新 DNS 缓存，重新点击检查更新即可
  • 配置描述文件方式：到「设置」-「隐私与安全性」-「描述文件」里把对应的 profile 移除，等一会儿就能重新看到 macOS 升级入口

写在最后#

以前我也喜欢追新，软件更到最新，经历过多次开发环境崩溃重建之后，逐渐对折腾失去了兴趣，自从 21 年入了 arm 的 mac 之后，过去4年，每隔1-2年就要有一次开发环境崩溃，我为自己过于激进导致的浪费时间感到后悔。

今年正式用上了 Time Machine，同时把更新彻底屏蔽了，再也不怕手抖了。这套方法我从 2025 年底一直用到 2026 年中，期间 Apple 推了不少次 macOS 26 的升级提醒，我这边一点动静都没有。

动动小手，五分钟，免除烦人更新小红点困扰
