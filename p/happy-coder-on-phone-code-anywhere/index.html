我把 Happy Coder 搬到手机上，随时随地写代码

用 Claude Code 写代码最大的痛苦就是得一直守在电脑前等确认。Happy Coder 让你用手机远程操控 Claude Code 和 Codex，终于可以离开工位了。
April 2, 2026  · eatmoreduck

起因#

最近在做一个小小的个人项目，日常开发基本靠 Claude Code + GLM 配合 Codex 做 vibe coding。说实话效率确实高，一个需求扔进去，它哐哐一顿写，我负责 review 和调整就行。

但有个问题一直困扰我——我得一直盯着它。

Claude Code 不是全自动的，它会频繁地要求你确认权限：这个文件能不能改？这个命令能不能执行？要不要继续？你要是不及时回应，它就停在那里等你。人根本走不开。尤其有些项目涉及到前端，我配置了 playwright mcp 做测试，在 claude 里已经配置"mcp__plugin_playwright*", 的情况下，仍然需要频繁的去请求权限，常常被确认的逻辑卡住，但又不想配置到用户目录的配置而导致权限放的过大。

我试过用手机远程连 Mac 的屏幕，想着至少能去倒杯水。但实际用起来太难受了，手机屏幕就那么大，终端界面的字小到看不清，点个按钮都得放大再放大，操作效率约等于零。

这个问题困扰了我挺久的，直到最近看到了 Happy Coder。

Happy Coder 是什么#

简单说，它就是 Claude Code 和 Codex 的手机客户端。

你在电脑上跑 Claude Code，通过 Happy Coder，你可以用手机实时看到它在干什么，并且能直接在手机上操作——确认权限、输入指令、切换任务，跟坐在电脑前一样。

几个我觉得比较关键的特点：

  • 免费开源，MIT 协议，代码在 GitHub 上
  • 端到端加密，你的代码和对话内容经过 AES 加密传输，中继服务器只搬运密文，看不懂你在传什么
  • 推送通知，Claude Code 需要你操作的时候会推送到手机
  • 支持语音，可以直接说话让它执行命令，不只是语音转文字那种
  • 多任务并行，可以同时跑好几个 Claude Code 实例

安全性聊聊#

看到"把代码传到手机上"这个说法，第一反应肯定是——安全吗？

我查了一下它的原理，大概是这样的：整个系统分三块——电脑端的 CLI、一个中继服务器（Relay Server）、手机端的 App。你的电脑和手机之间不直接通信，所有数据都经过中继服务器转发。

但关键是，中继服务器只搬运加密后的数据块，它自己解不开。

具体来说：

  • 电脑端把 Claude Code 的终端状态用 AES 加密后上传
  • 手机端拉取加密数据，本地解密后展示
  • 你在手机上的操作指令反过来，也是加密后上传，电脑端解密执行
  • 密钥在你扫码配对的时候直接在两个设备之间交换，不经过服务器

这叫零信任架构——你不需要信任中继服务器的运营者，因为它从头到尾只接触密文。而且整个项目是开源的，加密实现可以自己审计。如果你还是不放心，Relay Server 也是开源的，完全可以自己搭一套。

对我来说，平时写代码传的是加密密文，比手机远程桌面明文传输终端画面安全多了。

安装配置#

整个过程不复杂，几分钟搞定。

电脑端#

先装 CLI 工具：

npm i -g happy-coder

装好之后直接运行：

happy claude

它会启动一个 Claude Code 的会话，同时把这个会话注册到 Happy 的服务上。你在终端里看到的跟正常用 Claude Code 没什么区别，只是这个会话现在可以被手机端接管了。

手机端#

去应用商店搜 “Happy Coder” 就行，iOS 和 Android 都有。

装好打开之后，它会引导你跟电脑端配对。

配对完成后，你就能在手机上看到电脑上正在跑的 Claude Code 会话了。

一个小技巧#

用了一段时间后发现一个问题——我老是习惯性地敲 claude 而不是 happy claude，等反应过来的时候已经晚了，会话没注册到 Happy 上。

后来我在 .zshrc 里加了个 alias：

alias claude='happy claude'

这样不管敲 claude 还是 happy claude，走的都是 Happy。如果某天 Happy 出问题了想用原版，敲 \claude 加个反斜杠就能跳过 alias。

实际使用体验#

用了几天，说说真实感受。

终于可以离开电脑了#

这是最大的变化。以前跑个任务，我得坐在那等它弹出确认框。现在手机收到推送，点开看一眼，确认就完事了。去厨房拿个外卖、沙发上躺着，都不耽误干活。

推送通知是真的好用#

Claude Code 需要你输入的时候、代码写完等你 review 的时候、出了什么问题的时候，都会推送通知到手机。这个功能看着不起眼，但实际用起来你会发现——终于不用一直盯着屏幕刷新了。

语音操作有点惊喜#

本来觉得语音操控 Claude Code 是个噱头，但实际用下来还不错。躺着的时候不想打字，直接说一句"继续"或者"执行"，它就动了。不是简单的语音转文字，是真的语音到动作的执行。

多任务是真的爽#

做新项目时的时候，经常前端后端同时改。现在我可以同时开两个 Claude Code 实例，一个改后端规则引擎，一个改前端配置页面，手机上切来切去看进度。以前得一个个排着来，现在并行搞效率高了不少，当然一个 claude 多个 agent 一次性下发任务命令执行也是非常爽的。

也有不完美的地方#

说说缺点。毕竟是第三方工具，界面上偶尔会有点卡顿。另外如果你家网络不太好，手机端的响应会有延迟，这个看你自己的网络环境。还有就是语音识别偶尔会抽风，说中文的时候准确率不太稳定（我感觉到了，所以我都用微信语音转文字）。

适合什么人#

如果你符合下面任意一条，可以试试：

  • 用 Claude Code 或 Codex 做主力开发工具，但不想一直坐在电脑前
  • 经常需要同时跑多个 AI 编程任务
  • 想在手机上也能随时查看和操控你的编程 session
  • 对代码安全有要求，需要端到端加密

写在最后#

Happy Coder 解决的不是什么大问题，它就是帮你把"必须坐在电脑前"这个束缚去掉了。但就是这个小变化，让整个 vibe coding 的体验舒服了很多。

至少对我来说，终于不用在等确认的时候干坐着了。
